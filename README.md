# IT Troubleshooting Guide
A collection of common IT issues and structured troubleshooting approaches used in real-world support environments.

##Scenario 1: WiFi Not Connecting

### Problem
User unable to connect to WiFi network

### Possible Causes
* Incorrect password
* DHCP issues
* Router problems
* Device network settings
  
## Root Cause Example
The issue was caused by the router failing to assign an IP address (DHCP issue), preventing the device from connecting to the network.

### Troubleshooting Steps
1. Verify WiFi is turned on
2. Forget and reconnect to the network
3. Restart the device
4. Restart the router
5. Check IP address assignment
6. Run network diagnostics

### Outcome
Device successfully connects to network

## Scenario 2: Slow Computer Performance

### Problem
Computer is running very slow

### Possible Causes
* Too many background applications
* Low storage space
* Malware or unwanted programs

### Troubleshooting Steps
1. Restart the computer
2. Check Task Manager for high usage
3. Disable unnecessary startup programs
4. Free up disk space
5. Run antivirus scan

### Outcome
Improved system performance

## Scenario 3: Printer Not Working

### Problem
Printer not responding or printing

### Possible Causes
* Connection issues
* Driver problems
* Printer queue stuck

### Troubleshooting Steps
1. Check printer power and connection
2. Restart printer and computer
3. Clear print queue
4. Reinstall or update drivers
5. Set printer as default

### Outcome
Printer resumes normal operation

##Troubleshooting Methodology
* Identify the problem clearly
* Analyze possible causes
* Test solutions step-by-step
* Confirm resolution
* Document the solution

## User Interaction
* Ask the user when the issue started
* Confirm if other devices were affected
* Guid the user step-by-step through basic checks

### Escalation
Escalate the issue if:
* Hardware failure is suspected
* Issue persists after all troubleshooting steps
* Network-wide outage is detected
